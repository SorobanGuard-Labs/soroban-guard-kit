# Contributor guide

Soroban Guard Kit is designed for contributors who can improve readiness one focused issue at a time.

## Good tasks

A good contributor task has a problem statement, files to review, expected behavior, acceptance criteria, and a clear maintainer review path.

## Security work

Do not publish exploit instructions for an untriaged vulnerability. Open a private maintainer discussion or follow the repository security policy when findings could affect deployed users.

## Pull requests

Keep PRs narrow. Include test output, screenshots for UI changes, and notes about any assumptions that still need maintainer judgment.
